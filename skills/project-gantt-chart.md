# Skill: 项目管理甘特图生成器

## 触发条件
当用户提供项目管理表格（Excel截图/图片/文本），要求生成甘特图、项目管理看板、进度可视化时触发。

## 输入
- 项目管理表格（图片或文本），通常包含：项目名称、阶段、任务项、优先级、负责人、目标完成时间、当前状态
- 可能有多个项目并行

## 输出
- 一个自包含的 HTML 文件（无需外部依赖），用 Canvas 绘制甘特图 + HTML/CSS 实现交互组件

## 核心能力

### 数据提取
1. 从用户提供的表格图片中识别：**阶段名称、任务名称、优先级(P0/P1/P2)、负责人、目标完成时间、实际完成时间、当前状态（已完成/进行中/未开始）**
2. 如果有多个项目，分别提取每个项目的数据
3. 推断时间范围：根据任务的起止时间确定甘特图的时间轴范围

### 生成规范

#### 数据结构
```javascript
const PROJECTS = {
  projectKey: {
    name: "项目名称",
    color: "#主题色",
    deadline: "2026-06-30",      // 上线截止日期
    deadlineLabel: "630",         // 简称
    tasks: [
      // 阶段行（可折叠）
      { phase: "1. 阶段名称", start: "起始日", end: "结束日", status: "phase" },
      // 任务行
      { task: "任务名称", p: "P0", owner: "负责人", start: "起始日", end: "结束日", status: "done|progress|todo" },
    ]
  }
};
```

#### 必须包含的功能（10项）
1. **多项目切换** — 顶部 tab 按钮，支持「全部项目」和单个项目视图
2. **进度环形图** — SVG 环形图展示每个项目的完成比例
3. **倒计时** — 顶部显示距各截止日期的剩余天数
4. **里程碑线** — 甘特图上用虚线标注关键截止节点，按项目智能显示（单项目只显示自己的里程碑）
5. **今日标记线** — 红色虚线标注当前日期
6. **今日待办面板** — 仅在「全部项目」视图显示，自动筛选进行中/应启动/即将启动的任务
7. **风险预警** — 逾期或即将到期的任务自动闪烁⚠️标记
8. **关键路径高亮** — 一键高亮所有 P0 未完成任务
9. **筛选与搜索** — 按负责人下拉筛选、按优先级筛选、关键词搜索
10. **阶段折叠/展开** — 点击阶段标题行折叠或展开子任务

#### 附加功能
- **鼠标悬停详情** — tooltip 显示任务完整信息（名称、负责人、优先级、时间、状态、剩余天数）
- **导出 PNG** — 一键将 canvas 导出为图片
- **深色模式** — 切换深色/浅色主题

#### 技术实现要点
- **Canvas 绘制甘特图**：左侧面板(任务名/优先级/负责人/状态) + 右侧时间轴(条形图)
- **HTML/CSS 渲染**：顶部 header、进度环形图、筛选工具栏、今日待办面板
- **交互**：通过 `rowRects` 数组记录每个条形的位置，实现 mousemove tooltip 和 click 折叠
- **风险闪烁**：用 `requestAnimationFrame` + `Math.sin` 实现 opacity 闪烁动画
- **里程碑智能显示**：每个里程碑关联项目列表，切换项目时过滤显示
- **自适应 DPR**：Canvas 使用 `devicePixelRatio` 确保高清屏清晰

#### 样式规范
- 状态颜色：✅已完成 `#43a047` / 🔄进行中 `#fb8c00` / ⏳未开始 `#1e88e5`
- 优先级：P0 红底 `#fce4ec` / P1 橙底 `#fff3e0` / P2 绿底 `#e8f5e9`
- 阶段汇总条：紫色 `#5e35b1`，两端菱形标记
- 今日线：红色 `#f44336` 虚线
- 里程碑线：橙/红色虚线 + 顶部标签
- 深色模式背景：`#1a1a2e` / 卡片 `#16213e`

#### 布局参数（可根据任务数量调整）
- `LEFT_W = 440` — 左侧面板宽度
- `DAY_W = 10` — 每天的像素宽度
- `ROW_H = 32` — 每行高度
- `HDR_H = 48` — 表头高度

## 执行步骤
1. 分析用户提供的表格图片/数据，提取所有项目和任务信息
2. 确定时间轴范围（最早开始日 → 最晚结束日 + buffer）
3. 确定里程碑节点（从上线目标日期推导）
4. 生成完整的自包含 HTML 文件
5. 用浏览器打开验证效果
